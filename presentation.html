Project Overview

The Attendance Management System (AMS) replaces error-prone manual registers with an automated, role-based digital platform. Employees mark their own attendance; managers get real-time visibility and reporting.

🎯 Problem Solved

  • Eliminates manual registers & spreadsheets
  • Prevents buddy-punching & time fraud
  • Provides instant attendance analytics
  • Automates late-arrival approval workflow
  • Enables data-driven HR decisions

🚀 Key Highlights

  • Secure JWT-based stateless authentication
  • Role-based access (Employee / Manager)
  • Real-time dashboards for both roles
  • CSV export for payroll integration
  • Docker-ready & CI/CD automated

JWT Security Implementation

Designed and implemented by Srinath Udhayakumar — stateless, scalable, token-based authentication using HMAC-SHA256 signing.

🔑 Token Structure

// JWT = Header.Payload.Signature // Header { "alg": "HS256", "typ": "JWT" } // Payload (claims) { "sub": "user@example.com", // email "role": "EMPLOYEE", // RBAC role "iat": 1710000000, // issued at "exp": 1710086400 // expires +24 h } // Signature: HMAC-SHA256 of above

⚡ Token Generation

public String generateToken( String email, String role) { return Jwts.builder() .subject(email) .claim("role", role) .issuedAt(new Date()) .expiration(new Date( System.currentTimeMillis() + jwtExpiration)) // 24 h .signWith(getSigningKey()) // HS256 .compact(); }

🔄 Authentication Flow

1

Login Request

Client sends POST /api/auth/login with { email, password }. Spring Security uses DaoAuthenticationProvider to validate credentials against the BCrypt hash stored in PostgreSQL.

2

Token Issued

JwtService.generateToken() creates a signed HS256 token containing the user's email and role. The token (valid 24 hours) is returned to the client in the response body.

3

Subsequent Requests

Client stores the token in localStorage and the Axios interceptor automatically appends Authorization: Bearer <token> to every request.

4

JWT Filter (JwtAuthenticationFilter)

Runs once per request (extends OncePerRequestFilter). Extracts the Bearer token, validates signature & expiry via JwtService, loads CustomUserDetails, and sets the SecurityContextHolder.

5

Role-Based Access Control

Spring Security's SecurityConfig restricts /api/employee/** to EMPLOYEE role and /api/manager/** to MANAGER role. Unauthorized access returns HTTP 403.

🛡️ Security Features

  • Stateless — no server-side session storage
  • BCrypt password hashing (adaptive, salted)
  • CSRF disabled (REST API — safe)
  • CORS configured for known origins only
  • Token expiry enforced on every request

⚙️ Security Configuration

  • /api/auth/** → Public (no token required)
  • /api/employee/** → EMPLOYEE role only
  • /api/manager/** → MANAGER role only
  • /api/attendance/summary → MANAGER only
  • All other endpoints → Authenticated

  • GET /api/employee/dashboard

  • GET /api/employee/attendance/today

👔 Manager

  • GET /api/manager/dashboard
  • GET /api/manager/attendance
  • GET /api/manager/attendance/today-status
  • POST /api/manager/attendance/{id}/approve-late
  • GET /api/attendance/summary
  • GET /api/attendance/today-status
  • GET /api/manager/export/csv

📌 API Design Principles

  • RESTful resource-based URLs with JSON request/response bodies
  • Consistent HTTP status codes: 200 OK, 201 Created, 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 409 Conflict
  • Global exception handler maps domain exceptions to proper HTTP responses
  • Pageable responses for list endpoints (page, size, total params)
  • Date range filtering supported on attendance history & CSV export endpoints
